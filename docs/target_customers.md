# 目标客户分层与核心痛点场景

本文档定义 News Summary 平台的优先客户分层、典型角色与核心痛点场景，用于指导产品功能与内容过滤的优先级。

---

## 一、优先客户分层（建议顺序）

### 1. 第一优先：投资 / 研究从业者

**细分角色**

| 角色 | 说明 | 典型场景 |
|------|------|----------|
| 买方（Buy-side） | 基金、资管、家族办公室等投资决策者 | 晨会前快速扫盘、持仓相关事件追踪 |
| 卖方（Sell-side） | 投行研究、券商分析师 | 写日报/周报、客户问询时快速检索多源 |
| 行业研究 | 咨询、智库、企业战略研究 | 跨行业热点扫描、竞品与产业链动态 |

**核心痛点**

- **信息过载**：每日海量新闻，难以在有限时间内（如 30 分钟内）抓住真正重要的变化。
- **多源演化**：同一事件在不同媒体、不同时间点的报道需要合并理解，缺少「事件→多源证据」的结构化视图。
- **信噪比**：大量公告、停牌、例行披露会稀释真正影响定价/决策的信号。

**期望体验**

- 每日固定时间（如早报）内看到「今天最重要的 N 件事」。
- 每个事件具备：发生了什么（3–5 条要点）、为什么重要（市场/政策影响）、多源链接可追溯。
- 可按主题（地缘、经济、市场、供应链、大宗、AI）快速导航，对应系统分类：`geopolitics`, `economy`, `markets`, `supply_chain`, `commodities`, `ai_tech`, `energy_infra`。

**与现有能力映射**

- 聚类 + 热度排序 → 事件级聚合与重要性排序。
- Digest 结构（要点 + 多源链接）→ 满足「抓重点 + 多源」。
- 主题过滤 → 满足按主题扫盘；需持续压制公告/停牌类噪音。

### 2. 第二优先：跨境业务与供应链管理者

**细分角色**

| 角色 | 说明 | 典型场景 |
|------|------|----------|
| 跨境业务负责人 | 进出口、海外运营、国际合规 | 贸易政策、制裁、关税、地缘风险对业务的影响 |
| 供应链 / 采购 | 采购总监、供应链规划、物流 | 断供风险、港口/航线、原材料价格、脱钩/近岸化 |
| 能源与基础设施 | 能源采购、电力/基础设施相关 | 能源价格、电力短缺、地缘对能源供应的影响 |

**核心痛点**

- **风险可见性**：地缘、贸易、能源、物流等风险分散在不同信源，缺少「事件 → 影响 → 可执行建议」的串联。
- **响应时效**：希望在新政策/冲突/灾害发生后，尽快得到「对供应链/成本/交付」的影响解读，而非仅看到新闻标题。
- **跨主题关联**：单条新闻可能同时涉及地缘 + 供应链 + 大宗，需要按「风险类型」或「业务影响」聚合。

**期望体验**

- 事件卡中明确：发生了什么、对供应链/成本/交付的可能影响、建议关注或动作（如备货、替代源、合规检查）。
- 订阅维度：主题（地缘、供应链、大宗、能源）、国家/地区、关键词（如某港口、某原材料）。
- 对应系统分类侧重：`geopolitics`, `supply_chain`, `commodities`, `energy_infra`。

**与现有能力映射**

- 主题过滤与分类 → 已覆盖地缘、供应链、大宗、能源；可扩展「影响级别」「地区标签」以支持「影响→动作」展示。
- Digest 统一结构 → 可作为「事件→影响→链接」的基础；后续可增加「影响摘要」或「动作建议」字段。

### 3. 第三优先：企业战略 / 公关 / 政府事务团队

**细分角色**

| 角色 | 说明 | 典型场景 |
|------|------|----------|
| 企业战略 | 战略部、BD、投资发展 | 行业趋势、政策动向、竞品与产业链新闻 |
| 公关与传播 | 公关、品牌、舆情 | 公司/品牌/高管相关报道、行业负面与危机预警 |
| 政府事务 | 政府关系、合规、公共政策 | 监管、立法、产业政策、国际关系对公司的影响 |

**核心痛点**

- **定向监控**：需要按公司名、行业、关键词（如某法规、某地区）做持续监控，而不是只看通用热点。
- **预警与时效**：重大监管/舆情/政策变化需要「即时提醒」或「高频摘要」（如每 2 小时），而非仅每日早报。
- **可审计与可回溯**：谁在何时因何原因被提醒、依据哪些信源，需要可追溯。

**期望体验**

- 「我的看板」：仅展示与己相关的公司/行业/关键词。
- 告警规则：阈值触发（如某公司名出现频次、某政策关键词出现）或重大事件即时推送。
- 历史检索：按时间、主题、关键词检索过往摘要与原文链接。
- 对应系统分类：在现有主题基础上，依赖「关键词/实体订阅」与「用户配置」，而非仅固定主题。

**与现有能力映射**

- 当前为「无账号、无订阅」的通用 digest；后续需增加：用户/团队、订阅配置（关键词、公司、行业、时间窗）、推送与告警、历史检索与导出。

## 二、核心痛点场景汇总

| 优先级 | 客户分层 | 核心痛点场景 | 产品侧重 |
|--------|----------|--------------|----------|
| P0 | 投资/研究 | 30 分钟内抓重点 + 同一事件多源演化 | 每日 Top N 事件卡、多源链接、主题导航、降噪 |
| P1 | 供应链/跨境 | 事件→影响→动作建议、风险可见性与响应时效 | 影响解读、地区/主题订阅、信噪比 |
| P2 | 企业战略/公关/政府事务 | 按公司/行业/关键词监控与预警、历史检索 | 订阅、告警、看板、检索与审计 |

## 三、与系统主题分类的对应关系

当前 `topic_filter` 中的分类与上述客户需求的对应关系：

| 系统分类 (Category) | 投资研究 | 供应链/跨境 | 企业战略/公关/政府事务 |
|--------------------|----------|-------------|--------------------------|
| geopolitics        | ✓ 地缘与政策 | ✓ 贸易与地缘风险 | ✓ 政策与政府关系 |
| economy            | ✓ 宏观与利率 | ✓ 成本与贸易环境 | ✓ 产业政策与宏观 |
| markets            | ✓ 市场情绪与估值 | — | ✓ 行业与竞品 |
| supply_chain       | ✓ 产业链与脱钩 | ✓ 核心 | ✓ 供应链与采购 |
| commodities        | ✓ 大宗与成本 | ✓ 原材料与能源成本 | ✓ 成本与资源 |
| ai_tech            | ✓ 科技与主题投资 | — | ✓ 行业与监管 |
| energy_infra       | ✓ 能源与电力 | ✓ 能源与电力风险 | ✓ 能源政策与基础设施 |

后续扩展功能（如订阅、告警、影响标签）时，可据此优先加强对应分类下的内容质量与展示结构。
